maven框架

为啥要使用 maven 框架

  • 整合的 jar 包的太多，需要维护，可能版本冲突问题
  • 每个 IDE 可能目录结构不一样，项目维护比较麻烦

maven组成部分

  • 项目对象模型：project object model
    • 根据 maven 中的 pom.xml 生成
  • 项目生命周期：project lifecycle
    • 对项目的编译，运行，打包，部署等管理
  • 依赖管理系统：dependence management system
    • 对项目所依赖的 jar 包进行管理

作用

  • 依赖管理
    • 对 jar 包进行管理
  • 统一项目目录管理
  • 项目管理
    • 编译，运行，打包，部署等

    • idea2020及以上 + maven3.6.3

maven 坐标

  • 概述
    • 用于在 maven 仓库确定要使用的 jar 包
    • 在确定公司，再确定项目，再确定版本
  • 组成
    • groupid：公司名称
    • artifactid：项目名称
    • version：版本号

9、maven 生命周期和插件

  • 概述
    • clean：清理
    • default：核心阶段
      • 编译，测试，打包，部署等
    • site：产生报告
  • default：
    • compile
      • 编译项目，将资源存储到 target 目录中
    • test
      • 单元测试
      • clean -> compile ->test
    • package
      • 对项目进行打包 （配置文件打包成什么方式，这边就是什么格式）
      • clean -> compile ->test ->package
    • install
      • 将 jar 或者 war 包安装到本地仓库，此时不同项目也可以使用
      • clean -> compile ->test ->package -> install
    • deploy
      • 将 war 或 jar 部署到服务器

依赖范围 scope

  • 概述

  • 通过 scope 属性来决定 jar 包的范围

  • 依赖范围

    • 主代码：main
    • 测试打包：test目录可用
    • 打包：运行时可用

  • scope

    | scope | 主代码 | 测试代码 | 打包 | 备注 |
    | ———- | ——— | ———— | —— | ————————————————- |
    | compile | ok | ok | ok | 默认值，log4j，dbutils |
    | test | | ok | | junit |
    | provide | ok | ok | | servlet-api |
    | runtime | | ok | ok | mysql（springboot时都是配置文件） |

依赖排除

  • 概述

    • 项目 A 依赖项目 B, 项目 B 依赖项目 C，且 A 不传递依赖 C，此时就需要排除依赖

依赖冲突问题

  • 概述

    • 依赖传递中，如果依赖了不同版本的相同构件，可能会造成冲突

  • 路径优先：路径越短，使用短的那个

  • 声明优先：路径相同时，配置靠前的覆盖靠后的

版本管理

  • 语法

    • 主版本：有重大功能更新
    • 次版本：有小幅功能更新，有大BUG修复
    • 增量版本：有小BUG修复
    • 里程碑版本：给内部开发人员看的
      • SNAPSHOT : 测试版本
      • RELEASE : 正式版本
