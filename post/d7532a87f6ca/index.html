git 概述

概述：

  • 版本控制工具

  • 多人协同开发

  • 代码备份

  • 代码审查

  • 版本还原

  • 历史追查

  • 版本记录

工作机制：

  • 临时存储：以便每次都得上传，即可以创建多个文件 add 然后再 commit

平台：

  • 内网：gitlab
  • 外网：github，gitee

集中式：

  • 缺点
    • 服务器单点故障
    • 容错性差

分布式 ：

  • git 是 分布式版本控制系统 ，简称DVCS
  • 仓库：
    • 本地：开发人员自己的电脑
    • 远程：远程服务器的 git 仓库
  • 操作：
    • clone：克隆，将远程代码 复制 到本地
    • push：推送，将本地代码上传到远程仓库
    • pull：拉取，将远程代码 下载 到本地仓库
    • commit：提交，将暂存区的代码提交到本地仓库

git 安装，环境搭建

下载地址：https://git-scm.com/

设置全局用户签名：

  • 签名作用：区别不同操作者身份，由此来确定该版本是由谁提交的

  • 这里的用户签名和github（或其他代码托管平台）的账号无任何关系

  • 命令：

  • — global 表示全局，即所有的 git 项目都会使用这个属性
